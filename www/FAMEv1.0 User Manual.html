Fire Analysis Module for Ecological values (FAME) v1.0

User Manual.

Introduction

This software provides a Fire Analysis Module for Ecological values.  It has been given the acronym FAME.  This is used throughout the document to refer to the module.  This manual should be read in conjunction with the document FAMEv1.0_Inputs_Outputs.doc which provides details on the structure of the input and output files required by the module, and the values calculated in the outputs.

The module allows for spatial analysis fire sequence information from an input file of individual dated fire footprints. This is then of this data with fauna habitat distribution models (HDM) and vegetation maps of Ecological Fire Groups(EFG) of vegetation with associated lookup tables  to allow calculation of the summary used in comparison of fire scenarios and outcomes of fire histories Tolerable fire interval status of vegetation, and changes in modelled abundance of vertebrate Fauna.

The module also incorporates the aspatial Vegetation growth stage optimisation tool written in R that was developed under a previous SCI-ARIER project.

It provides a graphical user Interface to a sever where the analysis takes place via a web browser, to facilitate exploration of fire scenario options for users with minimal experience with command line or script-based analyses.

Workflow

The workflow consists of five processes.  One preparatory stage on a local PV and three components on the remote server.

Fire scenario analysis

1.       Combine the future fire scenario with past fire history and clip the data to the region of interest.  (Processing in ArcMap v10.3 or later on a personal computer).

2.       Upload this dataset to the FAME server and carry out Fire History analysis.

Fauna relative abundance (RA) following fire.

3.       Combine the outputs of 2. And EFG mapping with HDMs for vertebrate fauna and lookups modelling the effect time since fire and EFG on the relative abundance of each species within the area identified as its potential range by the HDM.

Tolerable fire interval (TFI) analysis.

4.       Combine the outputs of 2. With Ecological Fire Group(EFG) mapping and TFI values for the EFGs to output summaries, and if desired raster maps of the annual status of vegetation age relative to TFI and identify those areas  where there are inter-fire intervals that mean an area has been(or will be)  burned one or more times in an interval below TFI (BBTFI).

Aspatial Growth Stage Optimisation for Fauna Species.

5.       A separate process provides for aspatial optimisation of growth stages for EFGs in a defined area, given the lookup of relative abundance with growth stage and EFG, and a list of species occurring in that vegetation.

Initial Fire Scenario Analysis

This analysis gives the unique sequence of fires at any location, all inter fire intervals fire types as a polygon dataset where each polygon has a unique fire sequence.  This processing was previously carried out by the DELWP analysis known as FIREHAT.  The analysis also maps time since fire for every year of the analysis into the same polygon dataset.

g.       The resulting dialog box requires 4 inputs

Downloading Results from the Module

x.       Downloading results in handled from the Download results box at the bottom right of the Utilities tab:

 
Click on the Select (files) to download button to open a download dialog box:



this opens a file browser window, which will display all the directories of results data on the server, plus a file called dummy.txt.   Each session of FAME creates a results directory named <YYYYMMDD_HHMM> (numerical date time to the nearest minute) when the session is opened.  All results are housed in this directory.

The Download Results box provides the results directory name for the current session for easy identification. 
If you wish to download all results from the current session select this directory in the file browser window along with the dummy.txt file ( this last step is necessary because the browser will only download a directory when a file is also identified for download  you will simply ignore the dummy .txt file after download).

Alternatively you can browse the contents of the individual download directories and select files and directories therein for download.

When finished selecting, press the select button ant the bottom right of the file browser window. 

 

y.       The Download results box will then display the list of files (or directories containing files) that you have selected for download.  To complete the process click the Download files button at the bottom of the box.  This will zip your selection and download to a file named output.zip in your local downloads directory.  From here it can be unzipped, and the contents examined.  Details of the structure and content of each of the files in the downloads directory is given in FAMEv1.0_Inputs_Outputs.doc

Additional utilities.

Draft species list

The Utilities page includes an option to create a draft species list  this also estimates the proportion of the species range within the area of interest. 

 

This is useful if you are unsure which species it may be appropriate to include in your analysis.  It should only however be considered as a starting point for a custom list  the proportion of the range is based on the number of cells of the 225m binary HDM for the species in the area of interest, and may be further restricted to those on public land only.  To run this tool select the previously uploaded polygon for the area of interest, or one of the LF_Regions, chose whether or not to calculate only the proportion occurring on public land, and press the Run draft species list Button.  This file can be downloaded using the download procedure described above.  You can then edit the Include and Make Raster columns as required to finalise your custom species list.  This should be renamed, and uploaded to the module for further processing.

EFG_AREAS and spp_EFG_LMU files

Two further files required to run the aspatial GSO calculations  the EFG_AREAS and spp_EFG_LMU files are also calculated by this Utility , press the Run Spp EFG LMU for region to create these files.

They are downloaded in the same way.  At the moment they may require manual editing or addition to the field names before use in the GSO module.
